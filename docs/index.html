Operational Machine Passport

A machine-centered platform that registers industrial assets, ingests telemetry over OPC UA and MQTT, builds living machine passports, and explains anomalies in a way operators can actually use.

What it is

A platform built around the machine, not just the dashboard

The goal is simple: connect a machine, understand its signals, monitor its behavior, and keep all of that context inside a living passport that remains useful for operation, maintenance and diagnosis.

Register machines

Keep a persistent registry of assets, machine profiles and protocol-specific connection details.

Monitor live behavior

Track machine health, risky signals and pipeline quality through one operational workspace.

Explain anomalies

Combine rules, rolling statistics and connector health to make diagnosis more understandable.

Why the passport matters

It is not only a UI. It is a living machine record.

The passport brings together identity, connectivity, semantic mappings, monitoring context, diagnosis, maintenance data and other lifecycle information so that the machine can be understood as a whole.

Application views

Three core screens that show the platform end to end

These views capture the central idea of the tool: one place to browse machines, inspect behavior and understand anomalies.

Machines

Machine registry and passport view

Each machine has its own operational passport with health, baseline, semantic coverage and machine-specific lifecycle sections. This is the starting point for understanding the asset.

Monitor

Live monitoring focused on what matters

The monitoring view prioritizes active anomalies, risky signals and pipeline health instead of overwhelming the operator with raw telemetry. It is designed to answer what needs attention right now.

Diagnose

Explainable diagnosis with protocol context

Diagnosis combines signal-level detector outputs with protocol and connector quality, making it easier to distinguish between a machine fault, an observability issue, or a mixed condition.

Quick Start

Run the project locally

The repository contains the UI, analytics service, industrial exporter and demo simulators for both OPC UA and MQTT. 

git clone https://github.com/CIGIP-UPV/Operational-Machine-Passport.git
cd Operational-Machine-Passport
docker compose up -d --build

# UI         http://localhost:4000
# Grafana    http://localhost:3000
# Prometheus http://localhost:9093

Project scope

The stack includes a machine registry, machine passport, analytics engine, multi-protocol ingestion and a public-facing interface for monitoring and diagnosis.
